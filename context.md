Requirements: Personal Memory Ingestion System (Phase 1)1. System VisionA local-first, background-service memory pipeline that converts a temporary text "Inbox" into a permanent, AI-organized "Knowledge Base" without manual filing or UI interaction.2. Technical ArchitectureOperating Mode: Persistent background process; starts on boot.Input Buffer: A single plain-text file (Desktop/inbox.txt).Storage Logic: Append-only (never overwrite existing data).State Management: An internal _memories_index.json (Registry) tracks known entities, file paths, and metadata.Connectivity: Offline-first. If APIs (Gemini/DeepSeek) are down, store raw content and defer processing.3. Ingestion Logic (The Pipeline)3.1 Inbox ConsumptionTrigger: Monitoring inbox.txt for "Save" events.Action: Read content $\rightarrow$ Store Raw Copy $\rightarrow$ Wipe inbox.txt immediately.Validation: Ignore empty files or whitespace-only saves.3.2 AI Processing & CommandsThe system processes text based on a hierarchy of instructions:Command Markers (Highest Priority):#fix: Correct grammar/clarity.#expand: Elaborate on the thought.#research: Perform web-search (if online).#raw: Skip all AI modification.@append <entity> / @new <entity>: Force file routing.@umbrella <folder>: Force folder placement.Autonomous AI (Fallback): If no markers exist, AI identifies the Entity and Umbrella using the Registry and content context.Guardrails: AI must ignore natural language that "looks" like a command unless it uses the # or @ prefix.3.3 Routing & StorageScenarioActionExisting EntityAppend to Memories/<Umbrella>/<Entity>.md with timestamp.New EntityCreate new .md file in appropriate folder; update Registry.AmbiguityEscalated to Human-in-the-Loop.4. Human-in-the-Loop (Conflict Resolution)When AI confidence is low or options are split:File: Write to _decisions.txt.Format: Present the thought + specific numbered options (e.g., 1. Append to X, 2. Create Y).Resolution: System monitors _decisions.txt. Once the user types DECISION: #, the system executes and clears the block.5. Data Integrity & RegistryThe Registry (_memories_index.json) is the source of truth.Fields: entity_name, folder, file_path, summary, aliases, timestamps.Summaries: AI generates/updates a 1-sentence description of the entity in the Registry to improve future routing.Maintenance:Merging: Supported (managed via manual command in _decisions.txt or equivalent).Deletion: Entities can be deleted (removes from registry and storage).6. Execution RulesZero Noise: No popups or notifications.Privacy: All API keys in .env.Reliability: * Multiple saves in rapid succession must be handled via a queue.If a Registry update fails, the system must retry to prevent "ghost" files.Logs: Maintain _logs/ingestion.log for audit trails (Entity chosen, confidence score, action).7. Non-Requirements (Phase 1)No Search/Retrieval UI.No Chatbot interface.No manual folder management.No cloud-only requirements.
